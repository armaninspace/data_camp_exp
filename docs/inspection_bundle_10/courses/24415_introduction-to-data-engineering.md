## Visible Curated Q/A Pairs

Q: What is apply knowledge in a real-world case study by performing and scheduling an etl process.?  
A: Applying knowledge in a real-world case study by performing and scheduling an ETL process means using what you've learned to extract, transform, and load data, and automate these steps in a practical scenario.

Q: What is cloud computing usage?  
A: Cloud computing usage in data engineering refers to using cloud-based platforms and services to store, process, and manage data, enabling scalable and flexible data workflows.

Q: What is database management (sql and nosql)?  
A: Database management (SQL and NoSQL) involves using different types of databases to store, organize, and manage data. SQL databases use structured tables, while NoSQL databases handle unstructured or semi-structured data.

Q: What is etl process implementation?  
A: ETL process implementation is the process of extracting raw data from various sources, transforming it into a usable format, and loading it into relevant databases for analysis and insights.

Q: What is identify and use core data engineering tools, including cloud computing and parallel computing.?  
A: Identifying and using core data engineering tools means being able to work with technologies like cloud computing and parallel computing, which are essential for managing and processing large amounts of data efficiently.

Q: What are job scheduling in data pipelines?  
A: Job scheduling in data pipelines refers to using scheduling frameworks to automate and manage when data processing tasks run, ensuring data workflows happen at the right times.

Q: What are parallel computing techniques?  
A: Parallel computing techniques involve processing data simultaneously across multiple processors or machines, which speeds up data processing and is a key part of a data engineer's toolkit.

Q: What is understand and apply the etl (extract, transform, load) process.?  
A: Understanding and applying the ETL (Extract, Transform, Load) process means knowing how to collect data from sources, change it into a usable format, and load it into databases for analysis.

Q: What is understand the differences between data engineers and data scientists.?  
A: Understanding the differences between data engineers and data scientists means knowing that data engineers build and maintain data systems and pipelines, while data scientists analyze data to generate insights.

Q: What is work with different types of databases, including sql and nosql.?  
A: Working with different types of databases, including SQL and NoSQL, means being able to use both structured and unstructured data storage systems to manage and access data.

### Overview

Learn about the world of data engineering in this short course, covering
tools and topics like ETL and cloud computing.

Get Started in Data Engineering

Are you curious about a career in data engineering but don’t know where to
start? Or perhaps you want more information on what data engineers do before
you take the next steps? This four-hour course is an introduction to data
engineering and the core concepts, techniques, and tools you need to
understand to do the job.

Learn Data Engineering Concepts and Techniques

You’ll start by learning the differences between a data engineer and a data
scientist (and how they work together) before finding out more about the
tools of the trade, specifically talking about cloud computing and parallel
computing. By the end of the second chapter, you’ll understand the
applications of SQL and NoSQL, using DataFrames, and why parallel computing
is so important.

Perform ETL in Hands-on Exercises

The ETL process is core to a data engineer’s workflow. You will learn how
data is extracted, transformed, and loaded to get it ready for analysis and
generating insights. At the end of the course, you’ll put all this knowledge
into practice by performing and scheduling an ETL process yourself using
real-world data.

Our exercises and interactive tests allow you to review and cement your new
knowledge, so you’re confident discussing and applying it once you’ve
received your Statement of Accomplishment.

This introductory course is part of a data engineering Track, which offers
you pathways to improve your understanding of data engineering and a clear
set of next steps to becoming a professional data engineer.

### Syllabus

1. Introduction to Data Engineering  
In this first chapter, you will be exposed to the world of data
engineering! Explore the differences between a data engineer and a
data scientist, get an overview of the various tools data engineers
use and expand your understanding of how cloud technology plays a
role in data engineering.

2. Data engineering toolbox  
Now that you know the primary differences between a data engineer
and a data scientist, get ready to explore the data engineer's
toolbox! Learn in detail about different types of databases data
engineers use, how parallel computing is a cornerstone of the data
engineer's toolkit, and how to schedule data processing jobs using
scheduling frameworks.

3. Extract, Transform and Load (ETL)  
Having been exposed to the toolbox of data engineers, it's now time
to jump into the bread and butter of a data engineer's workflow!
With ETL, you will learn how to extract raw data from various
sources, transform this raw data into actionable insights, and load
it into relevant databases ready for consumption!

4. Case Study: DataCamp  
Cap off all that you've learned in the previous three chapters by
completing a real-world data engineering use case from DataCamp! You
will perform and schedule an ETL process that transforms raw course
rating data, into actionable course recommendations for DataCamp
students!
